Camera Sites

 

The camera sites are an important data element, not just the camera ID. The location is essential and if the camera is moved then we need a new camera id for the new location. The Browning cameras do not have a GPS and do not store any metadata of position, bearing or other information. This will have to be created using another APP. Perhaps a Garmin handheld or a phone using TrapNZ. We recommend using TrapNZ since it is used for all other monitoring, baits and traps by us at PFK.

 

 

QGIS

The camera location ID needs to be in a standard format for ease of classifying the images. CAM02, CAM02,... This is picked up by the app which will have a point feature layer containing the camera details and coordinates. This is currently in QGIS as a layer called camera_loc. Since QGIS can read TrapNZ records using WFS we can collect the field locations in future.

 

 

TrapNZ

The points are also in TrapNZ as a monitoring station camera type in the Kauri Point Halo Project. The intention is to use TrapNZ to record the location of the camera if it is moved and give it a new camera id, the next serial number.  

 

TrapNZ does allow uploading of images for classification, but not videos and the interface is limited but progressing. It may be a way of archiving useful images. The plan is to not use TrapNZ for archiving images or videos unless it is upgraded.
